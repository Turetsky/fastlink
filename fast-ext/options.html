Pair with the cloud relay

Pairing lets claude.ai on the web drive this browser. One click signs you in to the relay and pairs this browser automatically — no codes to copy. You only pair once per browser; you can pair multiple browsers (Claude drives whichever was opened or paired most recently).

✓ This browser is paired with the relay.

How to pair

  1. Click “Sign in & connect” below. A relay sign-in window opens (sign in with Google). On success this browser is paired automatically and a device token is saved here — there are no codes to copy. You only do this once per browser.
  2. No window opened, or pairing a second Chrome profile? Open “Have a pairing code instead?” further down, get a one-time code on the relay page, and paste it into the field.
  3. Add FastLink in claude.ai. Go to Settings → Connectors → Add custom connector, paste https://relay.ytx.app/mcp, save, then click Connect and sign in with the same account.
  4. Start driving. Open a chat in claude.ai and ask Claude to use FastLink on your active tab.

Advanced control …

FastLink works out of the box with DOM-based reading, clicking, and form-filling. Advanced control is an on/off setting (default on) that enables precise coordinate clicks/typing (vision tier), running scripts on the page, and capturing tabs that aren't in front (so Claude can drive a background tab while you stay on the chat). Turn it off to limit FastLink to DOM-based reading and form-filling.

This is a setting, not a Chrome permission grant — toggling it doesn't add or remove any browser permission.

Broker slot optional

Leave this on Primary unless you run FastLink in a second Chrome profile on the same machine. Each slot uses its own broker port (Primary: 9876, Secondary: 9877) so two profiles don't fight over one port. Set the second profile to Secondary.

Switching reloads FastLink to reconnect on the new port.

Notifications

Get a desktop alert when Claude finishes a task or appears to be stuck, so you don't have to watch the tab.

Updates

FastLink checks GitHub for new versions every few hours. When auto-apply is on, it reloads itself to the new version automatically as soon as it's available — no clicking required. Turn it off to just get the toolbar banner and a notification instead.

Site permissions

Per-origin consent for the cloud relay. Each site Claude touches can be set to Allow (read & act), Read-only (read but don't act), or Block. Changes sync to the relay; removing a row clears the local record.
