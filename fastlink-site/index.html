How it works

Three moving parts. Your browser stays yours.

FastLink is a Chrome MV3 extension plus a thin transport layer. Claude sends an instruction; the extension performs it on the page in front of you and streams the result back.

01 — IN YOUR BROWSER

The Chrome extension

An MV3 extension drives the active tab: snapshot the DOM, click, fill, navigate, scroll, and read console & network — robust even on heavy SPAs.

02 — ON YOUR MACHINE

Local WebSocket broker

For Claude Code & Desktop, a tiny local broker bridges the MCP server to the extension over WebSocket — one broker can fan out to several Claude sessions at once.

03 — FOR THE WEB

Multi-tenant cloud relay

So claude.ai on the web can reach you, a multi-tenant relay (Cloudflare Workers + Durable Objects) routes to your own browser — never a server’s.

claude → mcp → broker / relay → extension → your tab → result back to claude

Capabilities

Built for real pages, not demos.

From plain forms to canvas apps, FastLink picks the right strategy — fast DOM by default, vision when the DOM isn’t enough.

Fast DOM automation

Click, fill, navigate, scroll and snapshot the live DOM in milliseconds. Engineered to stay responsive on heavy single-page apps instead of freezing the tab.

clickfillnavsnapshot

Vision tier

When there’s no clean DOM target, FastLink locates by screenshot and does one-shot form-fills — working on canvas, iframes and non-DOM UIs that defeat selectors.

locatefill-visioncanvasiframes

Scout pre-reads

A fast page pre-read summarizes structure and intent up front, so Claude spends fewer round-trips finding its target.

Watch it work live

A side panel and on-page overlay mirror Claude’s transcript and current action across all your tabs — in real time.

Consent & control

Per-site allow / read-only / block, plus one-click pause and disconnect. Nothing happens on a site you didn’t green-light.

Console & network capture, resilient on heavy pages

FastLink reads console logs and network activity so Claude can debug what it’s driving — and its DOM strategy is tuned to keep working on dynamic, JS-heavy apps where naive automation hangs or times out.

Trust & safety

Your browser. Your session. Your call.

FastLink doesn’t spin up a headless server with your cookies. It acts in the browser already in front of you — and you stay in the loop for every action.

It’s your real browser

No cloud profile, no exported credentials. Claude uses the session you’re already signed into — locally.

You see every action

The overlay and transcript narrate each step as it happens. There’s no hidden activity.

Per-site consent

Set allow, read-only or block for each site. Defaults are conservative; you opt in.

Pause & disconnect

One click stops Claude mid-task or cuts the link entirely. You’re always one tap from taking the wheel.

Get started

Up and running in three steps.

Install the extension, connect your Claude surface, and pick which sites it can touch. DOM mode works out of the box.

1

Add FastLink to Chrome

Install the MV3 extension and pin it. The toolbar badge shows your connection state.

2

Connect your Claude

Run the MCP server for Code or Desktop, or link relay.ytx.app for claude.ai on the web.

3

Set consent & go

Choose allow / read-only / block per site. Add a free Gemini key to unlock vision & scout.
